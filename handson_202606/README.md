# Databricks Lakehouse 入門ハンズオン（202606）

ニュース電子版（デジタル購読）サービスのデータを題材に、UC → Lakeflow Jobs → Genie Space を体験するハンズオン。
想定時間 **100〜120分**。詳細は上位フォルダの `企画書_handson.md` を参照。

## 進行順（受講者）

| # | ノートブック | 形式 | 内容 |
|---|---|---|---|
| 00 | `00_env` | 各自設定 | カタログ（固定）＋自分のスキーマを設定。最初に開く |
| 01 | `01_setup` | 実行のみ | データ生成 → UC に **bronze_* 4テーブル**を直接登録（Volume/CSV は使わない） |
| 02 | `02_explore_uc` | ハンズオン | UC探索（bronze_*）＋⭐AI説明文サジェスト＋簡単な集計 |
| 03a | `03a_silver` | コード確認のみ | bronze → silver_*（型変換・JOIN）。**ここでは実行しない** |
| 03b | `03b_gold` | コード確認のみ | silver → gold_* マート3種。**ここでは実行しない** |
| 03c | `03c_create_job` | ハンズオン | ⭐Lakeflow Job をUIで手動作成。**Job 実行で初めて silver→gold が走る**（2タスク） |
| — | （口頭） | 講義 | UC の Lineage / Insights / Quality 紹介 |
| 04 | `04_analysis` | ハンズオン | ⭐churn等を自力分析 → 手こずる体験 |
| 05 | `05_genie_space` | ハンズオン | 同じ問いを Genie で → payoff |
| 99 | `99_extra_cost` | 持ち帰り | system テーブルでコスト分析（任意） |

## 講師の事前準備

1. **共有カタログを作成**：代表者が `handson_202606`（または任意名）を作成し、受講者にスキーマ作成権限を付与
   - カタログ名を変えた場合は全ノートブックの `00_env` の `catalog` を合わせる
2. **サーバレス / SQL Warehouse** を有効化（Genie・Job 用）
3. **1アカウントで通しリハーサル**（特に Job 手動作成・Genie 作成・`system.billing` 権限）
4. スライド：イントロ、UC機能紹介、ビジネスユーザー展開事例

## データに仕込んだ「気づき」（検証済み）

| フック | 見え方 | どこで気づく |
|---|---|---|
| テクノロジーの 2025H2 読了率急落（67%→24%、他カテゴリは横ばい） | `gold_monthly_engagement` 折れ線 | 02 / 04 / 05 |
| 30代の課金転換率が突出して低い（2.6% vs 他8〜9%） | `gold_paywall_conversion` | 04 / 05 |
| 20代以下・2025年後半に解約が集中 | `silver_subscriptions`（churn_ym） | 04 / 05 |

`random.seed(42)` 固定のため、誰が実行しても同じデータになります。

## ノートブック形式

Databricks ソースノートブック形式（`.py`）。ワークスペースへ取り込む場合：
- **Git フォルダ / Repos** にこのフォルダごと取り込むのが簡単
- もしくは各ファイルをワークスペースにインポート（`.py` のまま自動でノートブック認識）
- 配布用に `.dbc` 化する場合は、ワークスペース上のフォルダを右クリック → エクスポート（DBC Archive）
